####Git 笔记-3 摘自:http://blog.csdn.net/jackystudio/article/details/12249419 #####Git的仓库的初始化: ``` shell #进入d盘目录 这里可以使用linux命令 因为此处用的是git自带的unix bash $ cd /d #创建JackyGit文件夹 $ mkdir JackyGit #进入JackyGit文件夹 $ cd JackyGit #显示当前所在路径 $ pwd #列出当前目录文件 $ ls #初始化当前目录作为Git仓库 $ git init Initialized empty Git repository in d:/JackyGit/.git/ #初始化完后,就可以创建目录文件了. #查看当前Git仓库的状态: $ git status # On branch master # # Initial commit # # Untracked files: # (use "git add <file>..." to include in what will be committed) # # JackyData02 # JackyData03 # JackyDir/ # Jackydata01 nothing added to commit but untracked files present (use "git add" to track) #增加文件到暂存区 #. 表示添加所有文件 (当有创建新的文件夹或文件时需要再次添加) $ git add . #此时再次查看当前git库状态 $ git status # On branch master # # Initial commit # # Changes to be committed: # (use "git rm --cached <file>..." to unstage) # # new file: JackyData02 # new file: JackyData03 # new file: JackyDir/JackyData.txt # new file: Jackydata01 #提交更改 $ git commit -m '提交信息' #“init AddFiles”是本次提交的注释，4个文件被改变，4个新插入 $ git commit -m "init AddFiles" [master (root-commit) aea0365] init AddFiles 4 files changed, 4 insertions(+) create mode 100644 JackyData02 create mode 100644 JackyData03 create mode 100644 JackyDir/JackyData.txt create mode 100644 Jackydata01 #再次查看当前git库状态 #在主分支上没东西可以提交了，工作目录是干净的 $ git status # On branch master nothing to commit, working directory clean ``` [注意:](javascript:0) 如果添加文件到暂存区时出现警告warning: LF will be replaced by CRLF in JackyData01。 原因： 回车(CR, ASCII 13, \r) ，换行(LF, ASCII 10, \n)。 这两个ACSII字符不会在屏幕有任何输出，但在Windows中广泛使用来CRLF（回车换行）标识一行的结束。而在Linux/UNIX系统中只有LF（换行符）。 使用git来生成一个工程后，文件中的换行符为LF， 当执行git add .时，系统提示：LF 将被转换成 CRLF。 解决方法： #移除.git文件夹 移除之前请确定代码 $ rm -rf .git #设置全局属性autocrlf为false $ git config --global core.autocrlf false #重新init git $ git init Initialized empty Git repository in d:/JackyGit/.git/
